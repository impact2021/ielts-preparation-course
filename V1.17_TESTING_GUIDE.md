# Version 1.17 Testing Guide

This document outlines how to test the new features and changes in version 1.17.

## Changes Summary

### 1. Version Update
- Plugin version updated from 1.16 to 1.17
- **Test**: Check plugin page shows version 1.17

### 2. Removed Passing Score Display
- Passing score is no longer shown on exercise frontend
- **Test**: 
  - Navigate to any exercise
  - Verify "Passing Score: XX%" is NOT displayed
  - Only "Number of Questions" should be visible

### 3. Auto-Navigation After Exercise Submission
- After submitting an exercise, the system automatically navigates to the next item after 5 seconds
- **Test**:
  - Submit an exercise within a lesson
  - Verify a countdown appears: "Automatically continuing in 5 seconds..."
  - Verify you can click "Cancel" to stop auto-navigation
  - Verify you can click "Continue" to navigate immediately
  - Verify after 5 seconds (if not cancelled), you're taken to the next item
  - If there's no next item, you should return to the lesson/course page

### 4. Enhanced Exercise Results Display
- Results now show question text, user's answer, correct answer, and feedback
- **Test**:
  - Submit an exercise with various question types
  - Verify for each question you see:
    - The question text
    - Your answer
    - The correct answer (if you got it wrong)
    - Any feedback configured for that question/option

### 5. Redesigned Multiple Choice Backend
- New structured interface for creating multiple choice questions
- **Test in WordPress Admin**:
  - Edit or create a new exercise (Quiz post type)
  - Add a new question or edit existing multiple choice question
  - Verify you see:
    - Individual option fields (not a textarea)
    - Checkbox next to each option to mark it as correct
    - Feedback field for each option
    - "Add Option" button
    - "Remove" button for each option (requires minimum 2 options)
  - Save the question and verify it saves correctly
  - Take the exercise on the frontend to verify it works

### 6. Backward Compatibility
- Existing exercises should continue to work
- **Test**:
  - Take an existing exercise that was created before v1.17
  - Verify it displays and functions correctly
  - Edit the exercise in admin
  - Verify old format is converted to new format for multiple choice questions

### 7. Documentation Cleanup
- 37 unnecessary .md files removed
- **Test**:
  - Verify only README.md, CHANGELOG.md, and SECURITY_SUMMARY.md remain in root directory

## Test Scenarios

### Scenario 1: New Multiple Choice Question
1. Create a new exercise
2. Add a multiple choice question with 4 options
3. Mark option 2 as correct
4. Add specific feedback for option 1: "This is incorrect because..."
5. Add specific feedback for option 3: "Almost! But consider..."
6. Save and take the exercise
7. Select option 1 - verify you see the specific feedback
8. Retake and select option 2 - verify you see correct answer feedback (if configured)

### Scenario 2: Navigation Flow
1. Create a lesson with:
   - Resource page 1
   - Exercise 1
   - Resource page 2
   - Exercise 2
2. Take Exercise 1
3. Submit answers
4. Verify auto-navigation takes you to Resource page 2
5. Click on Exercise 2
6. Submit answers
7. Verify auto-navigation takes you back to lesson page (no more items)

### Scenario 3: Cancel Auto-Navigation
1. Take any exercise in a lesson
2. Submit answers
3. Click "Cancel" on the countdown message
4. Verify countdown stops
5. Verify you can review results without being redirected
6. Click "Continue" manually when ready

## Expected Results

All tests should pass with the changes functioning as described. If any issues are found:
1. Check browser console for JavaScript errors
2. Check WordPress debug log for PHP errors
3. Verify WordPress and PHP versions meet requirements
