# Version 2.30 Testing Guide

## Pre-Testing Setup
1. Ensure WordPress environment is running
2. Activate IELTS Course Manager plugin (version 2.30)
3. Have admin access to test cloning functionality
4. Prepare test data (courses with lessons, sub-lessons, exercises)

## Test 1: Version Verification
**Objective:** Verify plugin version is updated correctly

**Steps:**
1. Go to WordPress admin → Plugins page
2. Locate "IELTS Course Manager" in the plugin list
3. Check version number

**Expected Result:**
- Plugin shows version 2.30

## Test 2: Question Type - Matching and Classifying Questions
**Objective:** Verify the renamed question type works correctly

### Test 2.1: Question Type in Admin
**Steps:**
1. Go to IELTS Courses → Add New Quiz (or edit existing)
2. Scroll to "Questions" meta box
3. Click "Add Question"
4. Check the "Question Type" dropdown

**Expected Result:**
- "Matching and classifying questions" appears in dropdown
- Should be listed after "Multiple Choice"

### Test 2.2: Question Creation
**Steps:**
1. Select "Matching and classifying questions" type
2. Enter question text
3. Add answer options (should show MC options interface)
4. Mark one option as correct
5. Save the quiz

**Expected Result:**
- Interface shows multiple choice options format
- Can add/remove options
- Can mark correct answer
- No errors on save

### Test 2.3: Frontend Display
**Steps:**
1. View the quiz on frontend
2. Locate the matching/classifying question

**Expected Result:**
- Displays with radio buttons (like multiple choice)
- All options are clickable
- Only one option can be selected at a time

### Test 2.4: Quiz Submission
**Steps:**
1. Select an answer
2. Submit the quiz
3. View results

**Expected Result:**
- Correct answer: Shows as correct with green feedback
- Incorrect answer: Shows as incorrect with red feedback
- Score calculated correctly (1 point per question)

### Test 2.5: Computer-Based Layout
**Steps:**
1. Create/edit a quiz with computer-based layout enabled
2. Add matching/classifying question
3. View on frontend

**Expected Result:**
- Question displays in two-column CBT layout
- Radio buttons work correctly
- Submission and scoring work as expected

## Test 3: Course Cloning - Basic Functionality
**Objective:** Verify basic course cloning works

### Test 3.1: Clone Button Visibility
**Steps:**
1. Go to IELTS Courses → All Courses
2. Create a new course (save as draft or publish)
3. Edit the course

**Expected Result:**
- "Clone Course" meta box appears in right sidebar
- Meta box shows description and "Clone Course" button

### Test 3.2: Clone Empty Course
**Steps:**
1. Edit a course with no lessons
2. Click "Clone Course" button
3. Confirm the dialog

**Expected Result:**
- Confirmation dialog appears
- Button changes to "Cloning..."
- Success message appears with link to cloned course
- Button re-enables after completion

### Test 3.3: Verify Cloned Course
**Steps:**
1. Click the link to edit cloned course
2. Check course details

**Expected Result:**
- New course exists as draft
- Title is original title + " (Copy)"
- Content matches original
- Settings/meta data preserved
- Course has no lessons (original was empty)

## Test 4: Course Cloning - With Content
**Objective:** Verify cloning preserves all course content

### Test 4.1: Setup Test Course
**Steps:**
1. Create a course with:
   - Course title: "Test Course"
   - Course description
   - Course category
   - 2-3 lessons
   - Each lesson with 1-2 sub-lessons
   - Each lesson with 1-2 exercises

### Test 4.2: Clone Full Course
**Steps:**
1. Edit "Test Course"
2. Click "Clone Course" button
3. Confirm dialog
4. Wait for completion

**Expected Result:**
- Success message appears
- Shows number of lessons cloned
- Provides link to edit cloned course

### Test 4.3: Verify Course Structure
**Steps:**
1. Open cloned course in editor
2. Check "Course Lessons" meta box

**Expected Result:**
- All lessons from original appear in clone
- Lesson order preserved
- Lesson titles unchanged

### Test 4.4: Verify Lesson Content
**Steps:**
1. Click edit link for first lesson in cloned course
2. Check "Lesson Content" section

**Expected Result:**
- All sub-lessons appear
- All exercises appear
- Order preserved
- Content intact

### Test 4.5: Verify Sub-Lesson Content
**Steps:**
1. Edit a sub-lesson from cloned course
2. Check content and settings

**Expected Result:**
- Content matches original
- Settings preserved
- Associated with correct lesson in clone

### Test 4.6: Verify Exercise Content
**Steps:**
1. Edit an exercise from cloned course
2. Check questions and settings

**Expected Result:**
- All questions present
- Question types preserved
- Answer options intact
- Correct answers marked
- Feedback texts preserved
- Reading texts included (if applicable)

### Test 4.7: Verify Original Unchanged
**Steps:**
1. Go back to original "Test Course"
2. Verify all content

**Expected Result:**
- Original course unchanged
- No data loss or corruption
- All associations intact

## Test 5: Course Cloning - Edge Cases

### Test 5.1: Clone Unsaved Course
**Steps:**
1. Click "Add New Course"
2. Enter title but don't save
3. Check sidebar

**Expected Result:**
- Meta box shows message: "Save the course first before cloning"
- No clone button available

### Test 5.2: Clone Large Course
**Steps:**
1. Create/use course with 10+ lessons
2. Each lesson with multiple sub-lessons and exercises
3. Clone the course

**Expected Result:**
- Cloning completes successfully
- May take longer but doesn't timeout
- All content duplicated correctly
- Status updates shown

### Test 5.3: Clone Multiple Times
**Steps:**
1. Clone a course
2. Clone it again
3. Clone it a third time

**Expected Result:**
- Each clone created successfully
- Each gets " (Copy)" suffix
- All are independent copies
- No interference between clones

### Test 5.4: Non-Admin User
**Steps:**
1. Log in as Editor or Contributor
2. Try to access course clone functionality

**Expected Result:**
- Users without proper permissions cannot clone
- Error message if attempted via AJAX

## Test 6: Integration Testing

### Test 6.1: Clone and Edit
**Steps:**
1. Clone a course
2. Edit cloned course title
3. Add new lesson to clone
4. Remove a lesson from clone
5. Modify an exercise in clone

**Expected Result:**
- All edits work normally
- Original course unaffected
- Clone behaves as independent course

### Test 6.2: Delete Original After Clone
**Steps:**
1. Clone a course
2. Delete original course
3. Check cloned course

**Expected Result:**
- Clone still accessible and functional
- All content intact
- No broken relationships

### Test 6.3: Publish Cloned Course
**Steps:**
1. Clone a course (created as draft)
2. Edit cloned course
3. Publish it

**Expected Result:**
- Course publishes successfully
- Appears in course listings
- All content accessible on frontend
- Students can enroll and access

## Test 7: Error Handling

### Test 7.1: Network Error Simulation
**Steps:**
1. Open browser developer tools
2. Set network to offline
3. Try to clone course

**Expected Result:**
- Error message appears
- Button re-enables
- No partial data created

### Test 7.2: Invalid Course ID
**Steps:**
1. Manually trigger AJAX with invalid course ID (via console)

**Expected Result:**
- Proper error message returned
- No PHP errors
- Graceful failure

## Test 8: Browser Compatibility
**Objective:** Verify clone functionality works across browsers

**Browsers to Test:**
- Chrome/Edge (latest)
- Firefox (latest)
- Safari (latest)

**Steps for Each:**
1. Login to WordPress admin
2. Edit a course with content
3. Click clone button
4. Verify success

**Expected Result:**
- Works identically in all browsers
- No JavaScript errors
- UI displays correctly

## Test 9: Performance Testing

### Test 9.1: Time Tracking
**Steps:**
1. Clone courses of different sizes:
   - Small: 1-2 lessons
   - Medium: 5-10 lessons
   - Large: 15-20 lessons
2. Note time taken for each

**Expected Result:**
- Small courses: < 5 seconds
- Medium courses: < 15 seconds
- Large courses: < 30 seconds
- No timeouts

## Success Criteria
✅ All tests pass without errors
✅ Cloned courses match originals exactly
✅ No data loss or corruption
✅ UI is responsive and user-friendly
✅ Error messages are clear and helpful
✅ Performance is acceptable

## Reporting Issues
When reporting issues, include:
1. Test number and name
2. Steps to reproduce
3. Expected vs actual result
4. Browser and version
5. WordPress version
6. Error messages or screenshots
