# Version 2.37 Testing Guide

## Overview

This guide provides comprehensive testing procedures for Version 2.37 of the IELTS Course Manager plugin, which adds inline input fields for summary completion questions.

## Pre-Testing Setup

### Verify Installation

1. Check plugin version in WordPress admin:
   - Navigate to **Plugins** page
   - Locate **IELTS Course Manager**
   - Verify version shows **2.37**

2. Verify plugin files:
   - Check `ielts-course-manager.php` line 6: `* Version: 2.37`
   - Check `ielts-course-manager.php` line 23: `define('IELTS_CM_VERSION', '2.37');`

3. Clear browser cache:
   - Hard refresh (Ctrl+F5 or Cmd+Shift+R)
   - Ensures new CSS loads properly

## Test Suite

### Test 1: Version Update Verification

**Objective:** Confirm version number is correctly updated

**Steps:**
1. Navigate to WordPress admin **Plugins** page
2. Locate IELTS Course Manager plugin
3. Check version number

**Expected Result:**
- ✅ Version displays as **2.37**

---

### Test 2: Create Inline Summary Completion Question

**Objective:** Test creating a new summary completion question with inline inputs

**Steps:**
1. Navigate to **IELTS Courses > Exercises**
2. Click **Add New** or edit an existing exercise
3. Click **Add Question**
4. Set **Question Type** to **Summary Completion Questions**
5. In the **Question** field, enter:
   ```
   Many people believe that [ANSWER 1] because of the importance of [ANSWER 2] in modern society.
   ```
6. In the **Correct Answer** field, enter:
   ```
   1:students|learners|2:education|learning
   ```
7. Save the exercise
8. View the exercise on the frontend

**Expected Result:**
- ✅ Question displays with text and two inline input fields
- ✅ Input fields appear where `[ANSWER 1]` and `[ANSWER 2]` were
- ✅ Input fields are approximately 200px wide
- ✅ Input fields are inline with the text (not on separate lines)
- ✅ Text is readable with adequate spacing

## Edge Cases

### Edge Case 1: Missing Answer Numbers

**Question:**
```
Text with [ANSWER 1] and [ANSWER 3] (skipping 2)
```

**Correct Answer:**
```
1:answer1|2:answer2
```

**Expected:**
- ⚠️ System should handle gracefully (may show error or ignore missing number)

### Edge Case 2: Duplicate Answer Numbers

**Question:**
```
Text with [ANSWER 1] and [ANSWER 1] again
```

**Correct Answer:**
```
1:answer1
```

**Expected:**
- ⚠️ Both inputs should accept the same answer

### Edge Case 3: Case Variations in Placeholder

**Question:**
```
Text with [answer 1] or [ANSWER 2] or [Answer 3]
```

**Expected:**
- ✅ All case variations recognized (regex is case-insensitive)

   - Measure page load time

## Security Testing

1. **XSS Prevention**
   - Try entering `<script>alert('xss')</script>` in an answer field
   - ✅ Should be sanitized/escaped in feedback

2. **SQL Injection**
   - Try entering `' OR '1'='1` in an answer field
   - ✅ Should not cause database errors

## Known Limitations

1. Input fields have fixed 200px width - may need adjustment for very long answers
2. Mobile screens may need special handling for very narrow widths
3. Question must have correct answer format (`N:answer`) for inline inputs to be checked properly

All tests should pass before deploying to production. Document any failures and fix before release.
