# Version 2.7 Testing Guide

This guide provides step-by-step instructions for testing all changes in Version 2.7.

## Overview

Version 2.7 includes:
1. Version update to 2.7
2. Grammar fix for singular/plural "point(s)" text
3. Fullscreen mode improvements for computer-based tests

## Prerequisites

- WordPress installation with IELTS Course Manager plugin activated
- At least one course with a computer-based test (CBT) exercise
- At least one quiz with questions having different point values (including 1 point questions)

## Test 1: Version Update

### Purpose
Verify that the plugin version has been updated to 2.7.

### Steps
1. Log in to WordPress admin
2. Navigate to **Plugins** page
3. Locate "IELTS Course Manager" in the plugin list

### Expected Results
- Plugin should display "Version 2.7"
- No errors should appear in the admin area

✅ **Pass** / ❌ **Fail**

---

## Test 2: Grammar Fix - Singular Point

### Purpose
Verify that questions worth 1 point correctly display "1 point" instead of "1 points".

### Steps
1. Create or edit a quiz (standard or CBT layout)
2. Add a question with exactly 1 point value
3. Save the quiz
4. View the quiz as a student (logged in)
5. Check the point display for the 1-point question

### Expected Results
- Question should display: "Question X (1 point)"
- NOT "Question X (1 points)"

✅ **Pass** / ❌ **Fail**

---

## Test 3: Grammar Fix - Plural Points

### Purpose
Verify that questions worth more than 1 point still correctly display plural "points".

### Steps
1. Create or edit a quiz (standard or CBT layout)
2. Add questions with 2, 3, or more points
3. Save the quiz
4. View the quiz as a student (logged in)
5. Check the point display for multi-point questions

### Expected Results
- Question should display: "Question X (2 points)", "Question X (3 points)", etc.
- Plural form should be used correctly

✅ **Pass** / ❌ **Fail**

---

## Test 4: CBT Fullscreen - Non-Fullscreen View

### Purpose
Verify that computer-based tests hide content when not in fullscreen mode.

### Steps
1. Navigate to a computer-based test (CBT layout) quiz
2. Access it through regular page view (not fullscreen)
3. Observe the page content

### Expected Results
- Quiz content (reading texts and questions) should NOT be visible
- A prominent notice should appear stating: "This computer-based test must be viewed in fullscreen mode for the best experience."
- A large "Open in Fullscreen" button should be displayed
- Quiz header, title, and description should still be visible

✅ **Pass** / ❌ **Fail**

---

## Test 5: CBT Fullscreen - Button Functionality

### Purpose
Verify that the "Open in Fullscreen" button works correctly.

### Steps
1. From a non-fullscreen CBT quiz view
2. Click the "Open in Fullscreen" button
3. Observe the behavior

### Expected Results
- A new window/tab should open in fullscreen mode
- The quiz content (reading texts and questions) should now be visible
- The "Open in Fullscreen" button should not be visible in fullscreen mode
- Layout should use full viewport height

✅ **Pass** / ❌ **Fail**

---

## Test 6: CBT Fullscreen - Layout Optimization

### Purpose
Verify that the fullscreen layout has been optimized with proper spacing.

### Steps
1. Open a CBT quiz in fullscreen mode
2. Examine the layout spacing
3. Check if you need to scroll down to see the question navigation

### Expected Results
- No unnecessary space below the computer-based-container
- Question navigation bar should be visible without scrolling
- Reading column and questions column should use full available height
- No excessive margins or padding causing wasted space
- The entire quiz interface should fit within the viewport

✅ **Pass** / ❌ **Fail**

---

## Test 7: CBT Fullscreen - Question Navigation

### Purpose
Verify that the question navigation buttons work properly in the optimized layout.

### Steps
1. Open a CBT quiz in fullscreen mode with multiple questions
2. Scroll through questions in the right column
3. Click various question navigation buttons at the bottom
4. Observe the scrolling behavior

### Expected Results
- Question navigation bar should remain visible at the bottom
- Clicking a question button should scroll to the corresponding question
- No need to scroll down to access the question navigation
- Navigation should be smooth and responsive

✅ **Pass** / ❌ **Fail**

---

## Test 8: Standard Quiz Layout (Regression)

### Purpose
Verify that standard (non-CBT) quizzes still work correctly.

### Steps
1. Navigate to a standard quiz (not CBT layout)
2. View the quiz as a student
3. Verify that questions display properly
4. Check point display (singular/plural)

### Expected Results
- Standard quiz layout should be unchanged
- Point grammar fix should be applied (1 point vs X points)
- No fullscreen restrictions for standard quizzes
- All quiz functionality should work as before

✅ **Pass** / ❌ **Fail**

---

## Test 9: Mobile Responsiveness (CBT)

### Purpose
Verify that the fullscreen improvements work on mobile devices or responsive views.

### Steps
1. Open a CBT quiz in fullscreen mode
2. Resize browser window or test on mobile device
3. Check layout responsiveness

### Expected Results
- Layout should adapt to smaller screens
- Question navigation should remain accessible
- No horizontal scrolling required
- Content should be readable and usable

✅ **Pass** / ❌ **Fail**

---

## Test 10: Browser Compatibility

### Purpose
Verify that the fullscreen functionality works across different browsers.

### Steps
1. Test CBT fullscreen mode in:
   - Chrome/Edge
   - Firefox
   - Safari
2. Test both opening fullscreen and using the quiz

### Expected Results
- Fullscreen mode should work in all major browsers
- Layout should be consistent across browsers
- JavaScript functionality should work properly
- No console errors

✅ **Pass** / ❌ **Fail**

---
