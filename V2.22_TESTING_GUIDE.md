# V2.22 Testing Guide

**Version**: 2.22  
**Date**: December 19, 2024

## Overview

This guide covers testing for two fixes:
1. Question navigation margin in fullscreen mode
2. No-answer feedback visibility for multiple choice questions

## Prerequisites

- WordPress admin access
- At least one course with quizzes
- Test student account
- Browser with developer tools (for CSS inspection)

## Test Suite

### Test Case 1: Question Navigation Margin - Visual Check

**Objective**: Verify that the question navigation bar has proper spacing in fullscreen mode

**Steps**:
1. Log in as admin
2. Navigate to a quiz with computer-based layout
3. Add the URL parameter `?fullscreen=1` to view in fullscreen mode
4. Scroll to the bottom of the page
5. Observe the spacing below the question navigation bar

**Expected Results**:
- ✓ Question navigation bar has visible spacing (20px) at the bottom
- ✓ The spacing is consistent with the design
- ✓ No elements are overlapping or too close together
- ✓ The layout looks professional and uncluttered

**How to Measure**:
1. Right-click on the question navigation bar
2. Select "Inspect Element"
3. Look for `.question-navigation` element
4. Check computed styles for `margin-bottom: 20px`

---

### Test Case 2: Question Navigation Margin - Non-Fullscreen Mode

**Objective**: Ensure the fix doesn't break normal (non-fullscreen) mode

**Steps**:
1. Navigate to the same quiz WITHOUT the `?fullscreen=1` parameter
2. Scroll to the bottom of the page
3. Observe the spacing

**Expected Results**:
- ✓ Question navigation bar has proper spacing
- ✓ Layout is not affected by the fullscreen changes
- ✓ All elements display correctly

---

### Test Case 3: No-Answer Feedback Field - Admin Visibility

**Objective**: Verify that the no-answer feedback field is visible for multiple choice questions

**Steps**:
1. Log in as admin
2. Go to WordPress admin → IELTS Courses
3. Edit an existing quiz or create a new one
4. Add a new multiple choice question or edit an existing one
5. Scroll down to the feedback section

**Expected Results**:
- ✓ "No Answer Feedback" section is visible as a separate box
- ✓ It appears below the options section
- ✓ It has a heading "No Answer Feedback"
- ✓ The textarea is editable
- ✓ The section is styled with a border and padding (consistent with other sections)

**Visual Checklist**:
- ☐ Heading says "No Answer Feedback"
- ☐ Label says "No Answer Selected Feedback"
- ☐ Help text says "Shown when the student submits without selecting an answer. HTML is supported."
- ☐ Textarea is present and editable
- ☐ Section has white background with border

---

### Test Case 4: No-Answer Feedback Field - Save Functionality

**Objective**: Verify that the no-answer feedback is saved correctly

**Steps**:
1. From Test Case 3, enter feedback text in the "No Answer Selected Feedback" field
2. Example text: "Please select an answer before submitting. This question requires a response."
3. Save the quiz
4. Refresh the page or navigate away and back
5. Re-open the quiz editor

**Expected Results**:
- ✓ The feedback text is preserved
- ✓ HTML formatting (if used) is maintained
- ✓ No data loss occurs

---

### Test Case 5: No-Answer Feedback - Student Experience (Standard Quiz)

**Objective**: Test that students see the feedback when they don't answer

**Steps**:
1. Log in as a test student
2. Open a quiz with multiple choice questions that have no-answer feedback set
3. Answer some questions but leave at least one multiple choice question unanswered
4. Submit the quiz
5. Review the results

**Expected Results**:
- ✓ Unanswered questions are marked as incorrect
- ✓ The "No Answer Selected Feedback" text appears in the feedback section
- ✓ The feedback is clearly associated with the question
- ✓ The feedback is readable and properly formatted

**Visual Checklist** (for standard quizzes):
- ☐ Question shows as incorrect (red indicator)
- ☐ Feedback appears in the "Question Feedback" section
- ☐ User answer shows as "(No answer provided)"
- ☐ Correct answer is displayed
- ☐ Custom no-answer feedback text is displayed

---

### Test Case 6: No-Answer Feedback - Student Experience (CBT Quiz)

**Objective**: Test the feedback display in computer-based test (CBT) quizzes

**Steps**:
1. Log in as a test student
2. Open a CBT quiz (computer-based layout) with multiple choice questions
3. Leave some multiple choice questions unanswered
4. Submit the quiz
5. Click "Review my answers" in the results modal

**Expected Results**:
- ✓ Unanswered questions show red border/highlighting
- ✓ Navigation buttons for unanswered questions are red
- ✓ Feedback appears directly below each unanswered question
- ✓ Feedback has proper styling (`.question-feedback-message` with red border)

**Visual Checklist** (for CBT quizzes):
- ☐ Question container has red left border
- ☐ Background is slightly reddish (#fef5f5)
- ☐ Feedback div appears at the bottom of the question
- ☐ Feedback has gray background (#f0f0f1) with red left border
- ☐ Feedback text is clearly readable

---

### Test Case 7: No-Answer Feedback - Other Question Types

**Objective**: Verify the field is visible for all question types

**Steps**:
1. In the quiz editor, create or edit questions of each type:
   - Multiple choice
   - True/False
   - Fill in the blank
   - Summary completion
   - Essay
   - Multi-select
2. For each type, check if the "No Answer Feedback" section is visible

**Expected Results**:
- ✓ "No Answer Feedback" section is visible for ALL question types
- ✓ The field is always in its own separate box
- ✓ The styling is consistent across all question types

---

### Test Case 8: Backward Compatibility - Existing Quizzes

**Objective**: Ensure existing quizzes work correctly

**Steps**:
1. Open a quiz that was created before this update
2. Check if existing feedback (if any) is still there
3. Submit the quiz as a student without answering some questions
4. Verify behavior

**Expected Results**:
- ✓ Existing feedback is preserved
- ✓ If no `no_answer_feedback` was set, it falls back to `incorrect_feedback` (as before)
- ✓ No errors or warnings appear
- ✓ All quiz functionality works as expected

---

### Test Case 9: General Feedback Fields - Multiple Choice

**Objective**: Verify that general feedback fields remain hidden for MC questions

**Steps**:
1. Edit a multiple choice question in the quiz editor
2. Look for the "Feedback Messages" section (the one that contains "Correct Answer Feedback" and "Incorrect Answer Feedback")

**Expected Results**:
- ✓ "Feedback Messages" section is HIDDEN for multiple choice questions
- ✓ This is correct because MC questions use per-option feedback
- ✓ Only the "No Answer Feedback" section is visible
- ✓ The per-option feedback fields (in each option) are visible

---

### Test Case 10: General Feedback Fields - Other Question Types

**Objective**: Verify that general feedback fields are visible for non-MC questions

**Steps**:
1. Edit a true/false or fill-in-blank question
2. Look for both feedback sections

**Expected Results**:
- ✓ "Feedback Messages" section is VISIBLE
- ✓ "No Answer Feedback" section is also VISIBLE
- ✓ Both sections are separate and clearly labeled
- ✓ All fields are functional

## Edge Cases

### Edge Case 1: Empty Feedback
**Test**: Leave the no-answer feedback field empty
**Expected**: No feedback is shown, but no errors occur

### Edge Case 2: HTML in Feedback
**Test**: Add HTML formatting (bold, italic, links) in the feedback
**Expected**: HTML is rendered correctly and safely (sanitized)

### Edge Case 3: Very Long Feedback
**Test**: Add a very long feedback message (500+ words)
**Expected**: Layout doesn't break, text wraps properly

### Edge Case 4: Special Characters
**Test**: Use special characters and emojis in feedback
**Expected**: Characters display correctly
