# Version 2.29 Testing Guide

## Overview

This guide provides comprehensive testing instructions for Version 2.29 of the IELTS Course Manager plugin, which updates matching questions to work like multiple choice questions.

---

## Pre-Testing Requirements

### Environment Setup
- WordPress 5.8 or higher
- PHP 7.2 or higher
- IELTS Course Manager plugin version 2.29
- Test data with matching questions (both new and legacy formats)

### Test User Accounts
- Admin account (for creating/editing questions)
- Student account (for taking quizzes)

---

## Test Cases

### 1. Plugin Version Verification

**Objective:** Verify that the plugin version has been updated correctly.

**Steps:**
1. Log in to WordPress admin
2. Navigate to Plugins → Installed Plugins
3. Locate "IELTS Course Manager"
4. Check the version number

**Expected Result:**
- Version should display as "2.29"

**Status:** ☐ Pass ☐ Fail

---

### 2. New Matching Question Creation (Admin)

**Objective:** Test creating a new matching question with multiple choice format.

**Steps:**
1. Log in as admin
2. Navigate to IELTS Courses → Quizzes
3. Create a new quiz or edit an existing one
4. Add a new question
5. Select "Matching" as the question type
6. Enter question text: "Match the capital city to the country: What is the capital of France?"
7. Verify the "Answer Options" section is visible
8. Add answer options:
   - Option 1: "London"
   - Option 2: "Paris" (mark as correct)
   - Option 3: "Berlin"
   - Option 4: "Madrid"
9. Save the question

**Expected Result:**
- The mc_options field should be visible for matching questions
- The correct answer field should be hidden
- Options can be added with "Add Option" button
- One option can be marked as correct
- Question saves successfully

**Status:** ☐ Pass ☐ Fail

---

### 3. Matching Question Display (Standard Layout)

**Objective:** Verify matching questions display with radio buttons in standard quiz layout.

**Steps:**
1. As a student, navigate to a quiz with a matching question
2. View the question

**Expected Result:**
- Question displays with radio button options
- Each option shows as a selectable radio button
- Options display the text entered in admin
- Only one option can be selected at a time
- Input type is radio, not text
- No text input fields are shown

**Status:** ☐ Pass ☐ Fail

---

### 4. Matching Question Display (Computer-Based Layout)

**Objective:** Verify matching questions display with radio buttons in CBT layout.

**Steps:**
1. Navigate to a quiz with computer-based layout enabled
2. View the same matching question
3. Check the display in the two-column layout

**Expected Result:**
- Question displays with radio button options in CBT layout
- Same radio button behavior as standard layout
- Options display correctly in the CBT interface
- Consistent with other multiple choice questions in CBT mode

**Status:** ☐ Pass ☐ Fail

---

### 5. Matching Question Answer Submission

**Objective:** Test submitting answers for matching questions.

**Steps:**
1. As a student, start a quiz with a matching question
2. Select an answer option (radio button)
3. Submit the quiz
4. View the results

**Expected Result:**
- Selected answer is submitted correctly
- Answer field name is `answer_{question_index}` (not `answer_match_...`)
- Scoring works correctly (1 point for correct, 0 for incorrect)
- Results page shows the submitted answer
- Correct answer is highlighted if answer was correct

**Status:** ☐ Pass ☐ Fail

---

### 6. Matching Question Scoring (Correct Answer)

**Objective:** Verify correct scoring for matching questions.

**Steps:**
1. Create a matching question worth 1 point (default)
2. As a student, take the quiz and select the correct answer
3. Submit and view results

**Expected Result:**
- 1 point awarded for correct answer
- Score calculation is correct
- Percentage calculation is correct
- Feedback shows "Correct!" or similar positive message

**Status:** ☐ Pass ☐ Fail

### 7. Matching Question Scoring (Incorrect Answer)

**Objective:** Verify correct scoring for incorrect answers.

**Steps:**
1. Use the same matching question
2. As a student, take the quiz and select an incorrect answer
3. Submit and view results

**Expected Result:**
- 0 points awarded for incorrect answer
- Score calculation is correct
- Percentage calculation is correct
- Feedback shows "Incorrect" or similar message
- Correct answer may be shown (depending on quiz settings)

**Status:** ☐ Pass ☐ Fail

### 8. Question Numbering

**Objective:** Verify matching questions are numbered as single questions.

**Steps:**
1. Create a quiz with:
   - Question 1: Multiple Choice
   - Question 2: Matching
   - Question 3: True/False
2. View the quiz as a student
3. Check question numbers

**Expected Result:**
- Questions are numbered sequentially: 1, 2, 3
- Matching question shows as "Question 2" (not a range)
- Point display shows "1 point" for matching question
- No sub-question numbering within matching question

**Status:** ☐ Pass ☐ Fail

### 9. Multiple Matching Questions

**Objective:** Test a quiz with multiple matching questions.

**Steps:**
1. Create a quiz with 3 matching questions in a row
2. Each question has 4 options with different correct answers
3. Take the quiz as a student
4. Answer each matching question
5. Submit and review results

**Expected Result:**
- All matching questions display correctly with radio buttons
- Each question is independent (selecting answer in Q1 doesn't affect Q2)
- All answers submit correctly
- Scoring is correct for each question
- Results show individual scores for each matching question

**Status:** ☐ Pass ☐ Fail

### 11. Admin Interface - Question Type Switching

**Objective:** Test switching between question types in admin.

**Steps:**
1. Create/edit a question
2. Select "Multiple Choice" type
3. Verify mc_options field is visible
4. Switch to "Matching" type
5. Verify mc_options field remains visible
6. Switch to "Short Answer" type
7. Verify mc_options field is hidden

**Expected Result:**
- Matching type shows mc_options field (same as Multiple Choice)
- Matching type hides correct_answer field
- Type switching updates the interface correctly
- No JavaScript errors occur during switching

**Status:** ☐ Pass ☐ Fail

### 12. Backward Compatibility - Legacy Matching Question

**Objective:** Test how legacy matching questions (with matches array) behave.

**Note:** This test requires existing data with old matching format or manual data creation.

**Steps:**
1. If possible, access a matching question created before v2.29
2. View it in admin
3. View it as a student
4. Note any issues

**Expected Result:**
- Legacy question may not display correctly without mc_options
- Admin should see mc_options field to allow conversion
- Clear indication that question needs updating

**Status:** ☐ Pass ☐ Fail ☐ N/A (no legacy data available)
